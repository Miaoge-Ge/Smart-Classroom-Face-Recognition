# 账号库（本地文档）

本目录用于记录系统中不同角色的账号信息，方便演示/交付时统一管理。

为避免泄露风险：
- 不要把真实密码写进可提交的文档里
- 真实密码仅写入 `accounts.local.yml`（该文件已被 `.gitignore` 忽略）
- 可提交的示例请写在 `accounts.example.yml`

## 角色说明

- 管理员（admin）：学生管理、课程管理、系统设置、权限管理等全部功能
- 教师（teacher）：查看本人授课课程的考勤记录与统计分析；可进行实时监控（需选择课程）
- 学生（student）：仅可查看“我的考勤”（需绑定学号到已存在的学生信息）

## 文件清单

- `accounts.example.yml`：账号记录模板（可提交/可分享）
- `accounts.local.yml`：本地真实账号与密码（不可提交）

## 建议流程

1. 管理员登录（默认账号：`admin/admin123`，首次上线建议立刻修改密码）
2. 管理员进入「课程管理」添加课程，并填写“授课教师”（建议填写教师账号的姓名/显示名）
3. 管理员进入「权限管理」创建教师/学生账号
   - 教师：建议“姓名/显示名”与课程的“授课教师”一致
   - 学生：必须填写学号（需先存在学生信息或已导入学生表）
4. 将真实密码保存到团队密码管理器或 `accounts.local.yml`（避免散落在聊天记录/截图里）
